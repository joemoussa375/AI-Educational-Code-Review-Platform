What Is This Project?

  • An AI-powered assistant that helps students write better code
  • Works like a virtual teaching assistant – available 24/7
  • Students submit their code → AI reviews it → provides helpful explanations
  • Doesn't just fix mistakes – teaches why something is wrong
  • Designed to run on regular laptops – no expensive hardware needed
  • Helps bridge the gap when human instructors aren't available

Problem Statement

  • Challenge: Rising student-to-faculty ratio limits personalized feedback
  • Issue with AI tools: Most optimized for industrial productivity, not education
  • Trust Gap: Students blindly trust AI → may learn incorrect practices
  • LLM Hallucinations: False code fixes, nonexistent libraries suggested
  • Goal: Educational code reviewer that teaches, not just fixes

Literature Review Overview

Four Key Themes (12 Papers)

  • Theme 1: LLMs in Computer Engineering Education
  • Theme 2: Limitations of Pure LLMs
  • Theme 3: Hybrid Architectures (Static + LLM)
  • Theme 4: Retrieval-Augmented Generation for Compliance
  • Core Tech: Transformer Architecture & Quantization

Literature: The Problem with AI Tools

Key Findings from Papers 1-4

  • 90 AI tools surveyed → Most prioritize speed over teaching
  • Junior developers accepted 20% of hallucinated suggestions blindly
  • Pure LLMs produce 35% more false positives than hybrid systems
  • Current tools generate code fast, but don't explain code well

What We Took

  • Need for educational focus over productivity focus
  • Importance of teaching why mistakes happen, not just fixing

Literature: The Hybrid Solution

Key Findings from Papers 5-7

  • Feeding static analysis to LLM improved fix rate from 50% → 85%
  • CodeLlama outperforms generic LLMs by 15% on code tasks
  • Static analysis catches 60% of vulnerabilities that humans miss

What We Took

  • Combine Static Analysis + LLM for best results
  • Use CodeLlama (code-specialized) instead of generic models
  • Static tools detect → LLM explains in educational language

Literature: RAG for Compliance

Key Findings from Papers 8-10

  • RAG improves rule compliance by ~40%
  • RAG-based comments are 20% more helpful to developers
  • RAG transforms AI from memory-based to reference-based

What We Took

  • Retrieve PEP 8 guidelines before generating feedback
  • Ground responses in actual documentation, not training memory
  • Reduces hallucinations and ensures style compliance

Literature: Core Technologies

Key Findings from Papers 11-12

  • 4-bit Quantization: Reduces memory by 75%, keeps 99% performance
  • Transformer Architecture: Self-attention enables understanding long code files

What We Took

  • Use 4-bit quantization to run on student laptops (~6GB VRAM)
  • Leverage Transformer-based CodeLlama for code understanding
  • Makes the system accessible without expensive hardware

Research Gap & Our Solution

Identified Gaps

  • Tools optimize for speed, not learning
  • Pure LLMs → hallucinations, no style enforcement (PEP 8)
  • Existing hybrid tools focus on security, not education

Our Solution: AI-Based Educational Code Reviewer

  • Hybrid Pipeline: Pylint (syntax) + RAG (guidelines) + CodeLlama (explanation)
  • Educational Focus: Explains errors, doesn't just fix them
  • Accessible: Runs on mid-range GPUs via 4-bit quantization
